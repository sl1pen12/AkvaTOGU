# Как запустить приложение АкваТОГУ локально

## Быстрый старт (Windows)

Дважды щёлкните по файлу:

**`start-all.bat`** — запустит backend и frontend в двух окнах.

После запуска откройте в браузере: **http://localhost:5173**

## Отдельные серверы

| Файл | Назначение |
|------|------------|
| `start-all.bat` | Backend + Frontend вместе |
| `start-backend.bat` | Только API (порт **5000**) |
| `start-frontend.bat` | Только интерфейс (порт **5173**) |
| `start-all.ps1` | То же, через PowerShell |

## Требования

- **Node.js** (для frontend): https://nodejs.org  
- **Python 3** (для backend): https://python.org  

При первом запуске скрипты сами установят зависимости (`npm install`, `pip install -r backend/requirements.txt`).

## Остановка

Закройте окна терминалов «АкваТОГУ Backend» и «АкваТОГУ Frontend» или нажмите `Ctrl+C` в каждом.

## Порты

- Frontend (Vite): `5173`
- Backend (Flask): `5000` — только для локальной разработки

## API в продакшене (мобильное приложение и обычный запуск)

По умолчанию приложение обращается к бэкенду на **Render**:

**https://pool-backend-nmuz.onrender.com**

Локальный Flask нужен только если вы явно задаёте переменную окружения перед сборкой:

`VITE_API_BASE_URL=http://localhost:5000`
